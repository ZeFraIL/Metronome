# Описание класса: MetronomeView

## 1. Общая информация
*   **Имя класса:** `MetronomeView`
*   **Тип:** Кастомное представление (наследует `View`)
*   **Назначение:** Этот класс отвечает за визуальную отрисовку метронома. Он рисует темный фон, шкалу темпов с музыкальными терминами и движущийся маятник с грузиком.
*   **Взаимодействие:** Он отправляет уведомления в `MainActivity` через интерфейс (`OnBpmChangedListener`) всякий раз, когда пользователь взаимодействует с маятником через касание экрана.

## 2. Переменные (поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `pendulumPaint` | `Paint` | Настройки «кисти» (цвет, толщина) для стержня маятника | `init`, `onDraw` |
| `weightPaint` | `Paint` | Кисть для скользящего грузика | `init`, `onDraw` |
| `scalePaint` | `Paint` | Кисть для линий шкалы | `init`, `drawScale` |
| `angle` | `float` | Текущий угол поворота маятника | `onDraw`, `setAngle` |
| `pendulumLength`| `float` | Рассчитанная длина маятника на основе размера экрана | `onSizeChanged`, `onDraw` |
| `weightY` | `float` | Вертикальное положение (высота) грузика на стержне | `onDraw`, `setBpm`, `bpmToY` |
| `currentBpm` | `int` | Значение скорости, отображаемое в данный момент | `setBpm`, `yToBpm`, `onTouchEvent` |
| `tempoMarkings` | `Map` | Словарь, сопоставляющий числа BPM с названиями (например, 120 = "Animato") | `static` блок, `drawScale` |

## 3. Методы класса

### Метод: `init`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:** Нет
*   **Что делает:** Настраивает все «кисти» (цвета и размеры текста). Это похоже на подготовку палитры перед началом рисования картины.
*   **Когда вызывается:** При первом создании представления (в конструкторах).

### Метод: `onDraw`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void`
*   **Параметры:** `Canvas canvas` («холст» для рисования)
*   **Что делает:**
    1.  Закрашивает фон темно-серым цветом.
    2.  Вызывает `drawScale` для отрисовки цифр и линий.
    3.  «Поворачивает» холст на текущий угол `angle`.
    4.  Рисует линию маятника и грузик в форме трапеции.
*   **Когда вызывается:** Каждый раз при вызове `invalidate()` (много раз в секунду во время анимации).
*   **Важно:** Этот метод должен работать очень быстро. Никогда не создавайте новые объекты (например, `new Paint()`) внутри `onDraw`!

### Метод: `onTouchEvent`
*   **Тип:** `public`
*   **Возвращаемое значение:** `boolean` (True, если касание обработано)
*   **Параметры:** `MotionEvent event` (детали о том, где пользователь коснулся экрана)
*   **Что делает:**
    1.  Определяет, нажимает ли пользователь на экран или ведет пальцем.
    2.  Рассчитывает новый BPM на основе координаты `Y` (высоты) касания.
    3.  Обновляет внешний вид и сообщает `MainActivity` об изменениях.
*   **Когда вызывается:** Когда вы касаетесь `MetronomeView` на экране.

### Метод: `yToBpm` и `bpmToY`
*   **Тип:** `private`
*   **Возвращаемое значение:** `int` / `float`
*   **Что делает:** Это методы-переводчики. `yToBpm` преобразует пиксельную позицию на экране в число BPM, а `bpmToY` делает обратное.
*   **Важно:** Они используют линейную интерполяцию (математический прием) для распределения значений между верхом и низом стержня маятника.

## 4. Жизненный цикл (Custom View)
Кастомные представления не имеют такого же жизненного цикла, как Activity, но у них есть:
*   **`onSizeChanged()`:** Вызывается при первом открытии или повороте экрана. Используется для измерения высоты маятника.
*   **`onDraw()`:** Сердце представления, где происходит сам процесс рисования.

## 5. Взаимодействие с интерфейсом (UI)
*   **Рисование:** Использует Canvas API для рисования линий (`drawLine`), текста (`drawText`) и путей (`drawPath`).
*   **Ввод:** Прямая обработка касаний через `onTouchEvent`.

## 6. Взаимодействие с другими компонентами
*   **Паттерн «Слушатель» (Listener):** Использует интерфейс `OnBpmChangedListener`. Это позволяет `MetronomeView` «крикнуть» сообщение вроде «Эй, BPM изменился на 140!», не зная точно, как работает `MainActivity`. Это называется **слабой связанностью** (Decoupling).

## 7. Общая логика
Представление является визуальным воплощением физического объекта. Оно рассчитывает, где должен находиться грузик в зависимости от BPM. Когда запущена анимация, `MainActivity` быстро меняет переменную `angle` и заставляет представление «перерисовать себя» (`invalidate`).

## 8. Упрощенное объяснение
**Аналогия:** Представьте, что `MetronomeView` — это **механическая кукла**.
*   Она знает, как выглядеть (фон, линии, цвета).
*   У нее есть «ниточки» (`angle` и `weightY`), которые управляют её движениями.
*   Когда вы прикасаетесь к ней и двигаете её руку (грузик), у нее появляется «голос» (`listener`), чтобы сообщить кукловоду (`MainActivity`), что что-то изменилось.

---
**Заметка для студентов:** Обратите внимание на блок `static`, где заполняется `tempoMarkings`. Использование `Map` — это эффективный способ хранения пар данных (BPM и Название) вместо длинного списка условий `if-else`.
