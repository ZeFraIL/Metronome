# Описание класса: MainActivity

## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** Activity (Активность)
*   **Назначение:** Это «мозг» приложения. Он управляет пользовательским интерфейсом, координирует работу визуального маятника и звука тиканья, а также контролирует общий жизненный цикл метронома (запуск, остановка и изменение скорости).
*   **Взаимодействие:** Он взаимодействует с `MetronomeView` для обновления визуальной анимации и получает от него данные, когда пользователь перемещает грузик маятника. Также он использует `SoundPool` для воспроизведения аудио и `ScheduledExecutorService` для обеспечения точного тайминга.

## 2. Переменные (поля класса)

| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `metronomeView` | `MetronomeView` | Ссылка на кастомный UI-компонент маятника | `onCreate`, `onBpmChanged`, `onProgressChanged` |
| `startButton` | `Button` | Кнопка запуска метронома | `onCreate`, `startMetronome`, `stopMetronome` |
| `stopButton` | `Button` | Кнопка остановки метронома | `onCreate`, `startMetronome`, `stopMetronome` |
| `tempoSeekBar` | `SeekBar` | Ползунок для настройки BPM (скорости) | `onCreate`, `setTempo`, `onBpmChanged` |
| `bpmTextView` | `TextView` | Отображает текущее значение BPM | `onCreate`, `setTempo`, `onBpmChanged` |
| `animator` | `ValueAnimator` | Отвечает за плавное движение маятника влево-вправо | `startMetronome`, `stopMetronome` |
| `soundPool` | `SoundPool` | Эффективно воспроизводит короткий звук «тика» | `onCreate`, `startMetronome`, `onDestroy` |
| `soundId` | `int` | ID загруженного звука тика в памяти | `onCreate`, `startMetronome` |
| `soundLoaded` | `boolean` | Флаг готовности звука к воспроизведению | `onCreate`, `startMetronome` |
| `currentBpm` | `int` | Хранит текущую скорость (ударов в минуту) | Во всем классе |
| `scheduler` | `ScheduledExecutorService` | Менеджер фоновых потоков для точного тайминга | `startMetronome`, `onDestroy` |
| `tickFuture` | `ScheduledFuture` | Ссылка на активную задачу тиканья (нужна для остановки) | `startMetronome`, `stopMetronome` |

## 3. Методы класса

### Метод: `onCreate`
*   **Тип:** `protected`
*   **Возвращаемое значение:** `void` (ничего)
*   **Параметры:** `Bundle savedInstanceState` (используется для восстановления состояния)
*   **Что делает:**
    1.  Устанавливает макет UI с помощью `setContentView`.
    2.  Связывает Java-переменные с элементами интерфейса в XML.
    3.  Инициализирует `SoundPool` для подготовки звука.
    4.  Настраивает слушатели кликов (Click Listeners), чтобы приложение знало, что делать при нажатии кнопок.
*   **Когда вызывается:** Автоматически при запуске приложения.
*   **Важно:** Здесь происходит вся настройка. Если компонент не инициализирован здесь, приложение может «вылететь» позже.

### Метод: `startMetronome`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:** Нет
*   **Что делает:**
    1.  Проверяет, не запущен ли уже метроном (чтобы не запустить его дважды).
    2.  Создает `ValueAnimator` для изменения угла маятника от -30 до +30 градусов.
    3.  Рассчитывает период (время между ударами) на основе BPM.
    4.  Использует `scheduler` для повторения звука «тика» через равные интервалы.
    5.  Отключает кнопку «Старт» и включает «Стоп».
*   **Когда вызывается:** При нажатии пользователем кнопки «Старт».
*   **Важно:** Тайминг обрабатывается в фоновом потоке, чтобы он оставался стабильным, даже если основной поток UI занят.

### Метод: `stopMetronome`
*   **Тип:** `private`
*   **Возвращаемое значение:** `void`
*   **Параметры:** Нет
*   **Что делает:**
    1.  Останавливает анимацию маятника.
    2.  Отменяет запланированные звуки «тика».
    3.  Возвращает маятник в центральное положение (угол 0).
    4.  Снова активирует кнопку «Старт».
*   **Когда вызывается:** При нажатии пользователем кнопки «Стоп» или при изменении BPM во время работы.

### Метод: `calculateDuration`
*   **Тип:** `private`
*   **Возвращаемое значение:** `long` (миллисекунды между ударами)
*   **Параметры:** `int bpm` (скорость)
*   **Что делает:** Использует математику (`60000 / bpm`), чтобы найти, сколько миллисекунд длится один удар.
*   **Когда вызывается:** Внутри метода `startMetronome`.

## 4. Жизненный цикл (Activity)
*   **`onCreate()`:** Вызывается в самом начале. Инициализирует всё необходимое.
*   **`onDestroy()`:** Вызывается при закрытии приложения. Здесь **обязательно** нужно освободить ресурсы (SoundPool) и остановить `scheduler`, чтобы приложение не тратило заряд батареи в фоновом режиме.

## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** `Button` (Старт/Стоп), `SeekBar` (управление темпом), `TextView` (отображение BPM) и `MetronomeView` (маятник).
*   **Связь:** Используется `findViewById(R.id...)` для связи дизайна XML с кодом Java.
*   **События:**
    *   `OnClickListener`: Обработка нажатий кнопок.
    *   `OnSeekBarChangeListener`: Обновление скорости при перемещении ползунка.
    *   `OnBpmChangedListener`: Кастомное событие, срабатывающее при перетаскивании грузика маятника.

## 6. Взаимодействие с другими компонентами
*   **MetronomeView:** `MainActivity` сообщает представлению значение BPM, а представление сообщает активности, если пользователь вручную изменил положение грузика.

## 7. Общая логика
1.  Пользователь устанавливает скорость (BPM).
2.  `MainActivity` рассчитывает, насколько часто должны быть щелчки.
3.  При нажатии кнопки «Старт» фоновый таймер (`scheduler`) начинает воспроизводить звуки.
4.  Одновременно с этим аниматор начинает раскачивать визуальный маятник на экране.

## 8. Упрощенное объяснение
**Аналогия:** Представьте, что `MainActivity` — это **Дирижер** оркестра.
*   Он смотрит в **партитуру** (настройку BPM).
*   Он говорит **барабанщику** (`SoundPool`), когда именно нужно ударить в барабан.
*   Он показывает **танцору** (`MetronomeView`), как быстро нужно двигаться.
*   Когда вы говорите «Стоп!», он заставляет всех замолчать и сесть на места.

---
**Заметка для студентов:** Обратите внимание, как `soundPool.release()` вызывается в `onDestroy`. В программировании это признак хорошего тона — всегда прибирать за собой ресурсы!
